# Whitelist Bypass 0.5.0-alpha.11

## Proxy и VPN

- Windows и Android получили заметный переключатель `VPN / Proxy` на основном
  экране;
- VPN использует системный TUN и направляет выбранный системной политикой
  трафик автоматически;
- Proxy не создаёт TUN и оставляет системные маршруты без изменений;
- локальный SOCKS5 доступен на `127.0.0.1:1080` по умолчанию;
- адрес, порт и учётные данные можно скопировать из клиента;
- режим сохраняется между запусками и блокируется на время активного
  соединения.

Proxy ускоряет сценарий с одним приложением за счёт исключения фонового
системного трафика, но само приложение должно поддерживать SOCKS5. Обычный
Speedtest без настройки прокси измеряет прямое соединение.

## Восстановление звонка

- Creator контролирует дедлайн установления нового PeerConnection после
  появления или замены участника;
- `disconnected` получает короткий grace period, `failed/closed` запускают
  восстановление сразу;
- после трёх неуспешных внутренних попыток Creator завершается, чтобы manager
  создал новый звонок и увеличил recovery generation;
- старый RelayBridge и оба KCP-контура закрываются при reset, поэтому прежние
  `wait_snd` строки больше не продолжают печататься после замены peer;
- панель показывает промежуточное peer recovery как degraded.

`wait_snd=2` сам по себе нормален: это число неподтверждённых KCP-сегментов, а
не код ошибки или число попыток восстановления.

## Android DNS

Private/carrier DNS из диапазонов `10.x`, `192.168.x`, `172.16/12`, CGNAT и
IPv6 больше не передаётся серверу как системный resolver. Такие адреса доступны
только внутри мобильной сети телефона. Клиент автоматически использует
публичные DNS по умолчанию, если системные адреса нельзя достичь с сервера.

## Обновление

1. В Portainer выберите
   `ghcr.io/sereza111/whitelist-bypass-portainer:v0.5.0-alpha.11`.
2. Сделайте pull/redeploy, не удаляя persistent `/data`.
3. Установите matching APK/EXE из GitHub Release.
4. Проверьте `0.5.0-alpha.11` в логах клиента и сервера.
5. Убедитесь, что у профиля включено `Автовосстановление`.
