# Android Joiner

Android-клиент собирается из тех же исходников relay, что Docker Creator и
Windows Joiner. В репозитории намеренно отсутствуют готовые mobile.aar,
librelay.so, APK и keystore.

Workflow **Build Android Joiner** выполняет:

1. тесты общего transport;
2. gomobile bind для Android VPN/tun2socks bindings;
3. cross-compile Go relay для armeabi-v7a и arm64-v8a;
4. debug-подпись branch-сборки или release-подпись tagged-сборки;
5. проверку подписи, ABI и SHA-256.

## Скачать

Для обычной проверки откройте GitHub **Actions → Build Android Joiner**,
выберите успешный run и скачайте artifact whitelist-bypass-android-....
После создания тега APK также прикрепляется к GitHub Release.

На Android может потребоваться разрешить установку приложений из браузера или
файлового менеджера. Обычные branch-сборки могут использовать временную
debug-подпись. Tagged-сборки `alpha.9+` требуют постоянный PKCS12 release key из
GitHub Secrets; CI проверяет SHA-256 сертификата перед публикацией APK.

Старую debug-signed `alpha.8` нужно удалить один раз из-за несовместимой
подписи. Подписанные `alpha.9+` устанавливаются поверх предыдущей версии без
удаления приложения и его профилей.

## Подключение

1. В панели Creator создайте отдельную сессию для телефона.
2. Добавьте выданную ссылку в Android-приложение.
3. Оставьте headless и Video; для VK приложение использует negotiated
   reliability auto.
4. Разрешите Android создать VPN-подключение.

Одна Creator-сессия рассчитана на один Joiner. Не подключайте ПК и телефон к
одной ссылке одновременно.

## VPN и Proxy

Переключатель **VPN / Proxy** находится на основном экране:

- **VPN** запускает Android `VpnService` и направляет весь либо выбранный Split
  Tunneling трафик через relay;
- **Proxy** не запускает `VpnService`, не меняет системные маршруты и поднимает
  authenticated SOCKS5 на `127.0.0.1:1080`;
- в Proxy-режиме через туннель идут только приложения, в которых SOCKS5 указан
  явно; обычный Speedtest и остальные приложения продолжают использовать
  прямое соединение.

Логин и пароль локального SOCKS5 копируются из клиента. Автоматический DNS
режим исключает недоступные private/carrier resolver адреса, чтобы запросы не
зависали после включения VPN.

## Раздать активный туннель с телефона на ПК

Android-клиент может отдать уже подключённый туннель другому устройству как
SOCKS5 по локальной сети. В этом режиме ПК не подключается к звонку сам:

```text
Windows apps -> Wintun -> tun2socks -> Android SOCKS5 -> Android tunnel -> Creator
```

1. Подключите Android-клиент к своей Creator-сессии.
2. Откройте **Settings → Proxy** и включите **Share SOCKS5 over LAN**.
3. Нажмите **Save**, переподключите туннель, снова откройте этот экран и нажмите
   **Copy PC configuration**. Автоматические логин и пароль сохраняются между
   перезапусками приложения.
4. ПК должен быть в той же доверенной Wi-Fi сети или подключён к точке доступа
   телефона.
5. В Windows Joiner включите **Route this PC through the Android phone**,
   вставьте скопированный блок и нажмите **Start**. Клиенту нужны права
   администратора для Wintun и маршрутов.

LAN-раздача по умолчанию выключена и никогда не работает без SOCKS5
авторизации. Не включайте её в публичной Wi-Fi сети. Если телефон становится
недоступен три проверки подряд, Windows-клиент снимает split-default маршруты,
возвращает обычный интернет и запускает штатный auto-reconnect.

Для этого сценария не нужна вторая Creator-сессия: Windows использует тот же
активный туннель Android. Однако прямой Android-клиент и обычный Windows Joiner
по одной call-ссылке одновременно запускать по-прежнему нельзя.

VK Video включает KCP только когда сервер подтвердил capability video_kcp1.
Со старым или raw сервером приложение остаётся в degraded raw режиме.
Исправление повторных DNS-запросов присутствует и в мобильном relay.
